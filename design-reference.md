# Design Reference — 점심 메뉴 추천 서비스 '점메추'

---

## 1. Design Principles

> 서비스 핵심은 "결정 피로도 제거". 디자인도 같은 방향이어야 한다.

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **3초 안에 결과로** | 진입 후 카드 확인까지 탭 3번 이내. 불필요한 단계·설명 제거 |
| 2 | **보여줄 것만 보여준다** | 카드에 들어가는 정보는 "선택에 필요한 것"만. 부가 정보는 숨김 처리 |
| 3 | **Tinder 방식으로 결정 피로 제거** | 한 번에 1장씩 집중. 좋으면 바로 지도 열기, 싫으면 다음으로. 선택 부담 최소화 |
| 4 | **음식처럼 따뜻하게** | 차갑고 딱딱한 UI보다 약간의 온기. 단, 과하지 않게 (앱이 아닌 도구) |
| 5 | **모바일 우선, 데스크탑은 별도 경험** | 모바일은 앱처럼 단계별 흐름. 데스크탑은 한 화면에서 필터+결과를 동시에 |

## 2. Color Palette

> 식욕을 자극하는 따뜻한 주황+스톤(warm gray) 조합. 라이트/다크 두 모드 모두 지원.

### 2-1. Primary (공통)
| 이름 | Hex | 용도 |
|------|-----|------|
| Primary | `#F97316` | CTA 버튼, 강조 텍스트, 선택된 칩 |
| Primary Hover | `#EA580C` | 버튼 hover / pressed 상태 |

### 2-2. Light Mode
| 토큰 | Hex | 용도 |
|------|-----|------|
| bg-base | `#FAFAF9` | 페이지 배경 (warm white) |
| bg-surface | `#FFFFFF` | 카드·패널 배경 |
| bg-muted | `#F5F5F4` | 비선택 칩, 입력 필드 배경 |
| bg-primary-subtle | `#FFF7ED` | 선택된 칩 배경, 강조 영역 |
| text-primary | `#1C1917` | 음식점명, 주요 정보 |
| text-secondary | `#78716C` | 거리, 가격대 등 보조 정보 |
| text-disabled | `#D6D3D1` | 비활성 상태 |

## 4. Spacing & Layout

### Spacing Scale (4px 단위)
```
4px  / 8px  / 12px / 16px / 20px / 24px / 32px / 48px / 64px / 96px
```

### 4-1. Mobile Layout
- **전체 구조**: 단계별 풀스크린 흐름 (Step 1 → 2 → 3)
- **max-width**: `100%` (full screen)
- **좌우 패딩**: `16px`
- **섹션 간격**: `24px`
- **하단 고정 버튼**: Safe area 포함 `padding-bottom: 24px + env(safe-area-inset-bottom)`
- **네비게이션**: 상단 최소 헤더 (로고 + 설정 아이콘)

```
┌──────────────────────┐
│  점메추         ⚙️   │  ← 헤더 (56px)
├──────────────────────┤
│                      │
│   [콘텐츠 영역]      │  ← 스크롤 가능
│                      │
├──────────────────────┤
│  [하단 고정 버튼]    │  ← 52px + safe area
└──────────────────────┘
```

### 4-2. Desktop Layout
- **전체 구조**: 랜딩 페이지(Hero) + 사이드바 필터 + 결과 영역이 한 화면에
- **max-width**: `1280px` (컨테이너), 중앙 정렬
- **좌우 패딩**: `48px`
- **섹션 간격**: `48px`
- **네비게이션**: 상단 수평 GNB (로고 | 서비스 소개 | 로그인)
- **앱 영역 레이아웃**: 좌측 필터 패널(320px) + 우측 결과 영역(flex-grow)

```
┌────────────────────────────────────────────────────┐
│  점메추                          [로그인] [시작하기]│  ← GNB (72px)
├────────────────────────────────────────────────────┤
│                                                    │
│   [Hero 섹션]                                      │  ← 랜딩 전용
│   "오늘 점심, 3초만에 정해드려요"                  │
│                                                    │
├─────────────────┬──────────────────────────────────┤
│  필터 패널      │  결과 영역                        │
│  (320px)        │                                  │
│  · 반경         │  [카드 1] [카드 2] [카드 3]       │  ← 3열 그리드
│  · 가격대       │                                  │
│  · 비선호 태그  │  [다시 추천받기]                  │
│  [추천받기]     │                                  │
└─────────────────┴──────────────────────────────────┘
```

### 4-3. Border Radius
| 컴포넌트 | Radius |
|----------|--------|
| 카드 | `16px` |
| 버튼 (Primary) | `12px` |

## 5. 핵심 컴포넌트 스펙

### 5-1. 결과 카드 (Restaurant Card)

**Mobile** — 가로 full width, 세로 스택
```
┌─────────────────────────────────┐
│  [카테고리 칩]          [거리]  │  ← Caption, text-secondary
│                                 │
│  음식점명                       │  ← Heading 2, text-primary
│  대표 메뉴 1 · 대표 메뉴 2     │  ← Body 2, text-secondary
│                                 │
│  ★ 4.3  (리뷰 128)   💰 1만원대│  ← Caption
│                                 │
│  [지도에서 보기 →]              │  ← Body 2, Primary, 우측 정렬
└─────────────────────────────────┘
```

**Desktop** — 3열 그리드, 세로로 더 넉넉하게
```
┌───────────────────┐
│  [카테고리 칩]    │
│                   │
│  음식점명         │  ← Heading 2
│  대표 메뉴 1      │  ← Body 2
│  대표 메뉴 2      │
│                   │
│  ★ 4.3 (128)      │
│  💰 1만원대       │
│  📍 350m          │
│                   │
│  [지도에서 보기 →]│
└───────────────────┘
```

- **Light**: 배경 `bg-surface(#FFF)`, 테두리 `1px solid border(#E7E5E4)`, 그림자 `shadow`
- **Dark**: 배경 `bg-surface(#1C1917)`, 테두리 `1px solid border(#44403C)`, 그림자 `shadow`
- 카드 간 간격: `12px` (mobile) / `20px` (desktop)
- 카드 내부 패딩: `16px` (mobile) / `24px` (desktop)

### 5-2. 필터 칩 (Exclude Tag Chip)

| 상태 | Light | Dark |
|------|-------|------|
| 비선택 | 배경 `#F5F5F4`, 텍스트 `#78716C` | 배경 `#292524`, 텍스트 `#A8A29E` |
| 선택 | 배경 `#FFF7ED`, 텍스트 `#F97316`, 테두리 `#F97316` | 배경 `#431407`, 텍스트 `#F97316`, 테두리 `#F97316` |

- 폰트: Caption(12px), 내부 패딩: `6px 12px`
- 칩 간 간격: `8px`, 줄바꿈 허용 (wrap)

### 5-3. Primary 버튼

| | Light | Dark |
|-|-------|------|
| 기본 | 배경 `#F97316`, 텍스트 `#FFF` | 배경 `#F97316`, 텍스트 `#FFF` |
| hover | `#EA580C` | `#EA580C` |
| 비활성 | 배경 `#D6D3D1`, 텍스트 `#FFF` | 배경 `#44403C`, 텍스트 `#78716C` |

- 높이: `52px`, 가로: `100%` (mobile full width) / `auto min-width:160px` (desktop)
- 폰트: Body 1 (15px) / Weight 600

### 5-4. 반경 선택 (Segmented Control)
```
[ 500m ]  [ 1km ]  [ 2km ]
```
| 상태 | Light | Dark |
|------|-------|------|
| 선택 | 배경 `#F97316`, 텍스트 `#FFF` | 배경 `#F97316`, 텍스트 `#FFF` |
| 비선택 | 배경 `#F5F5F4`, 텍스트 `#78716C` | 배경 `#292524`, 텍스트 `#A8A29E` |
| 컨테이너 | `#F5F5F4` | `#292524` |

### 5-5. 다크모드 토글 버튼
- 위치: GNB 우측 (데스크탑) / 헤더 우측 (모바일)
- 아이콘: ☀️ (라이트) ↔ 🌙 (다크)
- 애니메이션: 아이콘 rotate 180deg + 색상 전환, 300ms

## 6. 인터랙션 & 애니메이션

> 빠른 서비스답게 애니메이션은 짧고 목적이 있는 것만.

| 상황 | 방식 | 시간 |
|------|------|------|
| 카드 진입 | fade-in + slide-up (아래→위 8px) | 200ms, ease-out |
| 버튼 tap/click | scale 0.97 | 100ms |
| 칩 선택/해제 | 색상 전환 | 150ms, ease |
| 로딩 (추천 중) | 카드 skeleton shimmer | - |
| 모바일 페이지 전환 | slide-left (스텝 진행) | 250ms, ease-in-out |
| 다크모드 전환 | 전체 색상 cross-fade | 300ms, ease |
| 데스크탑 카드 hover | border-color Primary + shadow 강조 | 150ms |

## 7. 상태별 UI 가이드

### 로딩 상태
- 카드 영역에 skeleton 카드 3개 표시 (shimmer 효과)
- shimmer 방향: Light → `#E7E5E4 → #F5F5F4` / Dark → `#292524 → #44403C`
- "주변 식당을 찾고 있어요..." 텍스트 (Body 2, text-secondary)

### 빈 상태 (결과 없음)
- 아이콘: 빈 그릇 또는 지도 핀 (Primary 컬러)
- 타이틀: "조건에 맞는 식당이 없어요" (Heading 3)
- 본문: "반경을 넓히거나 필터를 줄여보세요" (Body 2)
- CTA: "조건 다시 설정" 버튼

### 에러 상태 (위치 거부)
- 타이틀: "위치 정보가 필요해요" (Heading 3)
- 본문: "현재 위치 기반으로 주변 식당을 찾아드려요.\n브라우저 설정에서 위치 권한을 허용해주세요." (Body 2)
- CTA: "위치 허용 방법 보기" (선택적)
